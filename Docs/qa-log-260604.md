# Giant QA Sweep — QA Log (2026-06-04)

This log captures findings from the four-pass QA sweep against commit `88d5ab13eafa180c68c39015c02eed06e5ac804f` (design system baseline + Pipeline normalization).

Findings live here until promoted (or kept as observations). Confirmed defects get appended to `FEATURE_BACKLOG.md` under `Bugs / Gaps` as `#BUG-260604-NNNN`.

## Method & Scope

- Live server probed on `http://127.0.0.1:4310/` (already running).
- Frontend audited by reading every source file under `web/src/` plus `web/index.html` and `web/src/styles.css`.
- API audited by curl/PowerShell against `/api/*` endpoints; test data created and cleaned up (two test jobs hidden via `not_a_fit + dead`; one test saved view deleted).
- Browser MCP was not available, so live screenshot/keyboard checks were inferred from code behaviour rather than live observation.
- Verification commands: `npm run web:typecheck`, `npm run web:lint`, `npm run web:build` — all pass (5 known react-refresh warnings).

## Severity Rubric

- **Critical** — data loss, app unusable, broken import/export, broken save, unrecoverable errors.
- **High** — core flow blocked or misleading, bad persistence, keyboard trap, major layout clipping on normal widths.
- **Medium** — inconsistent behavior, confusing state, weak responsive handling, visual hierarchy problem that slows use.
- **Low** — copy polish, minor alignment, small spacing/token inconsistency.
